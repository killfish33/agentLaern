## 二、核心代码导读（先看这 4 个文件）

### 1) `src/server/prisma/client.ts`

**职责**：创建 Prisma 单例（服务端统一 DB 客户端）。

- 从 `DATABASE_URL` 读取连接串。
- 使用 `PrismaPg` 作为 adapter。
- 开发环境挂载到 `globalThis` 复用实例。

你后续改动数据库访问时，默认都应通过这里导出的 `prisma` 进入。

---

### 2) `src/server/repositories/chat-repository.ts`

**职责**：会话与消息的统一数据访问入口。

当前方法：

- `ensureSession`：有 `sessionId` 就复用，不存在则创建。
- `createMessage`：写入消息（支持状态/seq）。
- `updateMessageContent`：更新 assistant 消息内容与状态。
- `listSessions`：按时间倒序取会话。
- `getSessionMessages`：按时间正序取某会话消息。

后续新增 API（会话列表/历史消息）建议直接复用这里的方法。

---

### 3) `src/app/api/chat/stream/route.ts`

**职责**：SSE 流式接口 + 消息落库。

当前流程：

1. 校验 query（`q/requestId/sessionId/cursor`）。
2. `ensureSession`（确保会话存在）。
3. 写入用户消息（`done`）。
4. 创建 assistant 消息（`pending`）。
5. 流式逐 token 输出并更新 DB（`streaming`）。
6. 完成后标记 `done`；异常或取消标记 `error`。

说明：当前仍是 Mock token 生成逻辑（`buildTokens`），但持久化链路已完整。

---

### 4) `src/app/page.tsx`

**职责**：前端流式聊天页面（基础版）。

已具备：

- 消息状态机（`pending/streaming/done/error`）。
- 连接状态（`connecting/reconnecting/streaming/...`）。
- 重连与去重基础（`seq` 去重、重连重试）。
- `sessionId` 接收与复用。

当前限制：

- 左侧“会话列表”还是静态假数据，尚未接入后端真实会话接口。
